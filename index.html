When running 
yarn run no-bug
error is thrown because in 
serve -s
launches the server in SPA mode and will respond to any path (and respond with 404)
When running 
yarn run bug
no error is thrown because in 
serve
tries to find the file on disk but fails and thus does not respond (I think)
Observe however how the network request is 404
On this page only one error is thrown when running 
yarn run bug
and two errors when running 
yarn run no-bug
Either way, expects two errors to be displayed in the console like this
